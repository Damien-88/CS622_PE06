# 4. Runtime Comparison of Simple Array vs Linked List

Comparison of the runtime of three basic operations; **insert**, **delete**, and **lookup** ; between two data structures; **simple array** and **linked list** ; for different sample sizes.

N = {10, 100, 1,000, 5,000, 10,000}


All insertions were performed at the **head** of the collection, and lookups were for the **last element**. Deletions were tested at different positions: first, middle, and last.


## 1. Observed Runtime Trends

| Operation           | Data Structure | Time Compl | Observed Behavior as N Increases 
-------------------------------------------------------------------------------------------------------------------------------------------------
| **Insert (head)**   | Array          | O(n)       | Slower for large N because inserting at index 0 requires shifting all elements. 
|                     |                |            | Very fast for small N. 
|                     |                |            |
|                     | Linked List    | O(1)       | Consistently fast regardless of N. 
|                     |                |            | Time grows very slowly as N increases. 
|                     |                |            |
| **Delete (middle)** | Array          | O(n)       | Deletion requires searching for the element (O(n)) and shifting elements after deletion. 
|                     |                |            | Slower for large N. 
|                     |                |            |
|                     | Linked List    | O(n)       | Deletion requires traversal (O(n)), but no shifting of elements is needed. 
|                     |                |            | Slightly faster than array for head deletion. 
|                     |                |            |
| **Lookup**          | Array          | O(1)       | Instant access. 
|                     |                |            | Runtime does not increase with N.
|                     |                |            | 
|                     | Linked List    | O(n)       | Traversal from head to last node takes longer as N grows. 
|                     |                |            |Lookup time grows linearly with N. 
--------------------------------------------------------------------------------------------------------------------------------------------------

## 2. Insights on Delete Operation Location

- **First element deletion**:
  - Linked list: O(1) -> fastest  
  - Array: O(n) -> requires shifting all remaining elements  

- **Middle element deletion**:
  - Both data structures require O(n) traversal/search  
  - Array slightly slower due to shifting elements  

- **Last element deletion**:
  - Linked list: must traverse entire list -> O(n)  
  - Array: direct access + pop -> O(1) if using `pop(-1)`  

- As sample size increases, these differences become more pronounced. 
    - For N ≥ 5,000, inserting at the head in an array becomes noticeably slower than in a linked list.
    - Lookup of the last element in a linked list becomes noticeably slower than in an array.


## 3. Summary of Performance

1. **Insertion**  
   - **Linked list** is superior for head insertion (O(1) vs O(n)).  
   - Array insertion at the front suffers from element shifting as N grows.

2. **Deletion**  
   - Both have O(n) complexity for middle/last deletion.  
   - Linked list is faster when deleting the first element.  
   - Array deletion involves shifting elements, so performance degrades with size.

3. **Lookup**  
   - Array provides O(1) access to the last element.  
   - Linked list requires traversal O(n), making it slower for large N.

4. **Sorting**  
   - Both implementations use **selection sort (O(n²))**, which dominates runtime for large N.  
   - Runtime increases rapidly with sample size, independent of data structure.


## 4. Key Takeaways

- **Linked lists** are ideal for **frequent head insertions or deletions at the front**, especially when N is large.  
- **Arrays** are ideal for **fast lookups**, particularly when accessing the last element.  
- For small N (≤ 100), the differences are negligible due to Python’s low-level optimizations.  
- As N grows, the choice of data structure has a significant impact on performance depending on the operation and the position of elements being deleted.
